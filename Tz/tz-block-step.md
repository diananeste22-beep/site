# ТЗ: Блок «Этапы работы» (Horizontal Scroll)

## 1. Концепция и UX-механика
Секция должна создавать ощущение «конвейера» или «технического процесса». 
* **Поведение:** При скролле вниз секция фиксируется (Pin), и контент внутри плавно перемещается горизонтально (слева направо).
* **Триггер:** Скролл-анимация (Scrollytelling).
* **Цель:** Показать последовательность работы как системный процесс, где каждый этап вытекает из предыдущего.

## 2. Структура (Layout)
Блок делится на две зоны:
1. **Статичная зона (Left Sidebar):** Заголовок и кнопка CTA. Они остаются на экране, пока пользователь листает этапы.
2. **Динамическая зона (Scroll Track):** Три карточки, которые движутся горизонтально.

## 3. Контент (3 этапа)

### 01. Аналитика
* **Заголовок:** Аналитика
* **Описание:** Постановка задач, сбор требований и фиксация концептуального вектора проекта

### 02. Проектирование
* **Заголовок:** Проектирование
* **Описание:** Разработка визуальных решений, поиск композиции и формирование стиля
### 03. Реализация
* **Заголовок:** Реализация
* **Описание:** Финальная полировка, техническая подготовка и передача исходников

## 4. Технические требования
* **Реализация:** GSAP (ScrollTrigger) для управления анимацией скролла. 
* **Верстка:** 
    * Использовать `flexbox` для контейнера с карточками.
    * `position: sticky` для левого блока.
* **Адаптив (Mobile):** 
    * Отключение горизонтальной анимации. 
    * Переход к вертикальному списку: карточки выстраиваются одна под другой с отступом 20-30px.
