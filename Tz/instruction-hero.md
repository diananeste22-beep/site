# ТЗ для Codex: Animated Hero Block для сайта Арсения

## Контекст проекта

Проект: Next.js сайт-портфолио графического дизайнера Арсения.

Текущий стек:
- Next.js
- React
- TypeScript / TSX
- Tailwind CSS
- `next/image`
- структура проекта через `src/app`

Главная страница находится здесь:

```txt
src/app/page.tsx
```

Текущую hero-секцию нужно заменить на новую анимированную hero-секцию. Остальные секции страницы не трогать.

Текущая фотография используется по пути:

```txt
/photo/i.png
```

Физически файл должен лежать в:

```txt
public/photo/i.png
```

---

## Главная задача

Создать cinematic animated Hero block для главной страницы сайта.

Hero должен при открытии сайта проигрывать короткую последовательную анимацию:

1. Сначала появляется текст:

```txt
Йо, я Арсений
```

2. Затем появляется большой жирный заголовок:

```txt
Делаю дизайн
```

3. После появления большой заголовок плавно смещается влево и становится основным hero-заголовком.

4. Справа плавно появляется фотография Арсения.

5. После этого появляется supporting line:

```txt
От UX до визуальных музыкальных релизов
```

6. В финальном состоянии hero должен выглядеть как полноценный первый экран сайта:
   - слева текстовый блок;
   - справа фотография;
   - есть понятный CTA;
   - композиция выглядит современно, жирно, дизайнерски.

---

## Важно по UX

Анимация должна быть короткой и не раздражать пользователя.

Ограничение по длительности:

```txt
2.3–2.8 секунды максимум
```

Нельзя делать длинное интро на 5–7 секунд.

Пользователь должен быстро понять:
- кто это;
- что он делает;
- куда нажимать дальше.

Hero не должен быть просто красивой заставкой. Он должен продавать услуги дизайнера.

## Визуальный стиль

Стиль: bold, clean, modern, street/editorial.

Hero должен ощущаться:
- уверенно;
- крупно;
- графично;
- с вайбом музыкальных релизов и дизайнерского портфолио;
- без перегруза.

### Цвета

Использовать текущую палитру сайта:

```css
background: #f1f0ec;
text: #111111;
accent: #2d63fc;
```

### Типографика

Заголовок должен быть очень крупным и жирным.

Рекомендации:
- uppercase;
- плотный tracking;
- line-height около `0.85–0.95`;
- сильный display-вайб.

Если в проекте уже подключены шрифты, не ломать их.

Если нужно предложить шрифт:
- для заголовков: `Unbounded`, `Russo One` или текущий жирный шрифт проекта;
- для текста: `Manrope`, `Inter` или текущий sans-serif.

Не подключать сторонние платные шрифты.

## Анимация

Использовать `framer-motion`.

Если его нет в проекте, добавить зависимость:

```bash
npm install framer-motion
```

### Sequence

Примерная логика анимации:

#### Шаг 1

`Йо, я Арсений`

- opacity: `0 → 1`
- y: `16px → 0`
- duration: `0.45s`
- delay: `0s`

#### Шаг 2

`Делаю дизайн`

- opacity: `0 → 1`
- scale: `0.96 → 1`
- y: `24px → 0`
- duration: `0.55s`
- delay: `0.45s`

#### Шаг 3

Main headline moves left into final position

- x: center → left
- duration: `0.65s`
- delay: `1.1s`
- easing: smooth, not bouncy

#### Шаг 4

Photo appears on the right

- opacity: `0 → 1`
- x: `40px → 0`
- scale: `0.96 → 1`
- duration: `0.65s`
- delay: `1.35s`

#### Шаг 5

Supporting line and buttons appear

- opacity: `0 → 1`
- y: `16px → 0`
- duration: `0.45s`
- delay: `1.75s`

Общая длительность анимации не должна превышать примерно `2.8s`.

## Responsive behavior

### Desktop

Desktop layout:

```txt
Left: text content
Right: photo
```

Требования:
- hero занимает минимум `100vh` или около этого;
- контент не должен прилипать к краям;
- максимальная ширина контейнера около `1360px`;
- фото справа должно быть крупным, но не ломать сетку.

### Mobile

На мобильных:

```txt
Text first
Photo below
```

Требования:
- не делать горизонтальный overflow;
- заголовок должен переноситься аккуратно;
- размер текста уменьшить;
- анимацию сделать мягче;
- фото не должно вылезать за экран.

---

## Accessibility

Обязательно учитывать `prefers-reduced-motion`.

Если у пользователя включено reduced motion:
- не проигрывать сложную sequence-анимацию;
- сразу показывать финальный layout;
- можно оставить только легкий fade-in или вообще без анимации.

Использовать `useReducedMotion` из `framer-motion`.

Пример:

```tsx
import { motion, useReducedMotion } from "framer-motion";
```

## Технические требования

1. Работать в файле:

```txt
src/app/page.tsx
```

2. Заменить только текущую hero-секцию.

3. Остальные блоки страницы не менять:
   - услуги;
   - портфолио;
   - этапы работы;
   - контакты;
   - навигацию;
   - footer.

4. Использовать существующий путь к фото:

```txt
/photo/i.png
```

5. Использовать `next/image`.

6. Не удалять `header`.

7. Не ломать `id="top"`, `id="portfolio"`, `id="contacts"`.

8. Не создавать ненужные новые страницы.

9. Не менять routing проекта.

10. Код должен быть валидным для Next.js App Router.

## Желательная структура кода

Можно вынести hero в отдельный компонент:

```txt
src/components/AnimatedHero.tsx
```

И подключить его в:

```tsx
import AnimatedHero from "@/components/AnimatedHero";
```

Потом в `src/app/page.tsx` заменить текущую hero-секцию на:

```tsx
<AnimatedHero />
```

Если в проекте нет alias `@`, использовать относительный импорт.

Если проще и чище, можно оставить компонент прямо в `page.tsx`, но лучше вынести отдельно.

## Tailwind-подход

Использовать Tailwind CSS, как в текущем проекте.

Пример классов по вайбу:

```tsx
<section className="relative isolate min-h-screen overflow-hidden bg-[#f1f0ec] px-5 pt-28 sm:px-8 lg:px-12">
  <div className="mx-auto grid w-full max-w-[1360px] items-center gap-10 lg:grid-cols-[1fr_0.85fr]">
    ...
  </div>
</section>
```

Для заголовка:

```tsx
<h1 className="text-[56px] font-black uppercase leading-[0.86] tracking-[-0.07em] text-[#111111] sm:text-[90px] lg:text-[132px]">
  Делаю дизайн
</h1>
```

Размеры можно корректировать, чтобы не ломалась адаптивность.

## Motion example

Можно использовать такой подход:

```tsx
const shouldReduceMotion = useReducedMotion();

const itemTransition = shouldReduceMotion
  ? { duration: 0 }
  : { duration: 0.6, ease: [0.22, 1, 0.36, 1] };
```

Для stagger/sequence можно использовать delays.

Не использовать слишком прыгающую анимацию. Нужна плавная, дорогая, уверенная анимация.

## Что нельзя делать

Не делать:
- длинный preloader;
- fullscreen-интро, которое блокирует сайт;
- мигающий текст;
- агрессивный glitch на весь экран;
- анимацию дольше 3 секунд;
- удаление важных секций страницы;
- замену всех стилей проекта;
- использование платных шрифтов;
- тяжелые видеофоны;
- зависимость от внешних CDN для критичных элементов.

## Acceptance criteria

Результат считается готовым, если:

- при открытии сайта проигрывается короткая hero-анимация;
- появляется `Йо, я Арсений`;
- появляется `Делаю дизайн`;
- заголовок плавно занимает левую часть hero;
- справа появляется фото `/photo/i.png`;
- появляется текст `От UX до визуальных музыкальных релизов`;
- есть две CTA-кнопки;
- ссылки ведут на `#portfolio` и `#contacts`;
- hero адаптивен на mobile/tablet/desktop;
- нет horizontal overflow;
- `prefers-reduced-motion` обработан;
- остальные секции страницы не сломаны;
- проект собирается без ошибок.
